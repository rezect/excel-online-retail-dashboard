# Суть проекта

Интерактивная система дашбордов для управления продажами, клиентами и ассортиментом e-commerce компании. Подробнее, у меня получились:

- Общий дашборд с прибылью (какие регионы в какие даты показали себя лучше всего)   
- Пользовательский дашборд (разделение пользователей на сегменты по поведению - дает понять, какие пользователи приносят больше денег при меньших вложениях, кого надо удержать, а кого наоборот - не трогать)
- Продуктовый дашборд (продакт аналитика по товарам - какие приносят 80% прибыли, какие еще 15%, а какие оставшиеся 5% - дает понять самые продаваемые товары. Также показывает проблемные товары, у которых больше всего отмен)

# Кому он полезен

Общий дашборд с выручкой и продуктовый дашборд будет интересен Product Manager'у и руководителю закупок (узнает в какое время и какие категории товаров самые прибыльные). Дашборд клиентов будет полезен маркетологу и менеджеру по работе с клиентами (они смогут узнать, каких пользователей надо удержать, а каких наоборот, не трогать. Также в каких регионах выгодно делать рекламу и продвигать нашу площадку).

# Какие проблемы решает

Было: аналитик вручную анализирует 500,000+ транзакций, для нового среза заново считает метрики.
Стало: данные подгружаются и обновляются метрики по одному клику
Результат: экономия времени аналитика, снижение риска ошибок ручных расчетов

# На каком уровне создавался

Создал проект для портфолио, чтобы продемонстрировать навыки работы с Power Query, DAX и построением многоуровневых дашбордов для бизнес-анализа.
